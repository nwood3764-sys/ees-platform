# EES Platform — Task List

Live task list. Updated as each commit lands. **Maintained by Claude** as part
of every working session — every completed step gets reflected here before
the next one starts.

## Format

- `[x]` = done and committed (commit hash in trailing parens)
- `[~]` = in progress (uncommitted changes on disk)
- `[ ]` = not started
- `[blocked]` = blocked on external dependency

**Self-referential hashes:** A commit can't know its own hash in advance, so
when a TASKS.md update lands in the same commit as the work it describes,
the entry uses `(this commit)` as a placeholder. A follow-up chore commit
runs `git log -1 --format=%H` and substitutes the real hash.

## Standing protocols

- **Documentation discipline:** every shipped feature that affects user behavior gets a help article in the same session — anchored to the relevant route, object, or concept so the in-app HelpIcon surfaces it in context. Don't go crazy; only document where users would intuitively reach for help. Required pieces per article: `ha_title`, `ha_summary` (search-friendly), `ha_body_markdown`, and at least one `help_article_anchors` row.
- **Migrations: production-only.** No Supabase dev branches. Always verify with a SELECT after `apply_migration`. Use `apply_migration` for DDL, `execute_sql` for reads.
- **Communication:** one yes/no question at a time. Make a recommendation, ask for approval. No multiple choice.

Drove a headless Playwright/Chromium instance at the live site to capture the actual runtime error: `ReferenceError: Cannot access '_' before initialization` thrown by `vendor-recharts-CIOW8Ti5.js` at line 9 col 16763. Root cause: a **circular import between vendor chunks**. `vendor-react-Dak0xn-W.js` first line: `import { d as C1 } from "./vendor-recharts-CIOW8Ti5.js"`. `vendor-recharts-CIOW8Ti5.js` first line: `import { c as Pi, g as oe, r as R, R as _ } from "./vendor-react-Dak0xn-W.js"`. Browser evaluation order: vendor-react starts → needs `C1` from vendor-recharts → vendor-recharts starts → needs `R` (React default) from vendor-react → vendor-react is still mid-evaluation, hasn't reached its export statement yet → `R` is in Temporal Dead Zone → throw. The shared symbol creating the back-edge is `fast-equals` (specifically its `deepEqual` export, minified to `dN`, re-exported as `d`). `@tiptap/react`'s `useEditor` hook uses `fast-equals` as its default equality function for `useSyncExternalStoreWithSelector`. `fast-equals` is also a transitive dep of `recharts` via `react-smooth`. When the previous Communications v1 commit added TipTap deps, Vite's `manualChunks` rule routed `@tiptap/*` into `vendor-react` (matching the implicit React-ecosystem fallback), but it had already assigned `fast-equals` to `vendor-recharts` (because recharts→react-smooth→fast-equals was already in the graph as a deeper recharts subtree). With TipTap now in vendor-react also needing fast-equals, vendor-react had to import the symbol from vendor-recharts — creating the cycle. Fix: rewrote `vite.config.js` `manualChunks` to explicitly route the editor stack into vendor-react (`@tiptap/*`, `prosemirror`, `orderedmap`, `rope-sequence`, `w3c-keyname`) AND explicitly route the shared utilities used by both ecosystems into vendor-react too (`fast-equals`, `use-sync-external-store`). Also added `victory-vendor` (recharts' d3-shape vendor) to the recharts side. Resulting chunk sizes: `vendor-react` 519.95 kB (was 432.53 kB), `vendor-recharts` 414.60 kB (was 420.87 kB), `RecordDetail` 433.44 kB (was 521.47 kB — TipTap moved out into vendor-react). All bundle hashes regenerated. Post-build verification: `vendor-react` chunk has zero imports from `vendor-recharts`; `vendor-recharts` imports from `vendor-react` (one-way, no cycle). Playwright/Chromium against the live deployed site after the push: zero pageerror events, React root mounted with 1 child, login form rendered with the expected EMAIL/PASSWORD/Sign In affordances. **Process failures this session**: (a) treated `npm run build` clean as sufficient signal to push — Vite does not error on undefined named imports, only the runtime does.
